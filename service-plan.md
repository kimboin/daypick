# 친구 일정 조율 사이트 기획서

## 1. 서비스 개요

### 서비스 한 줄 설명
친구들끼리 초대코드를 공유해 같은 날짜 범위 안에서 가능한 일정을 입력하고, 모두가 겹치는 날짜를 빠르게 찾는 일정 조율 웹서비스.

### 해결하려는 문제
- 단체 채팅에서 날짜를 하나씩 물어보며 약속 날짜를 정하는 과정이 번거롭다.
- 누가 어떤 날짜가 가능한지 한눈에 보기 어렵다.
- 여러 명이 동시에 입력해도 결과를 자동으로 정리해주는 도구가 필요하다.

### 핵심 가치
- 초대코드만으로 빠르게 방 생성 및 참여 가능
- 방장이 날짜 범위를 먼저 정해서 입력 범위를 제한
- 겹치는 날짜를 자동으로 추출
- 전체 후보와 개인별 가능 날짜를 함께 확인 가능

## 2. 주요 사용자

### 1) 방장
- 약속 방을 생성하는 사용자
- 6자리 숫자 초대코드를 발급받음
- 본인 닉네임 입력
- 약속 가능한 날짜 범위 설정
- 본인의 가능한 날짜 입력
- 친구들에게 초대코드 전달

### 2) 참여자
- 초대코드를 전달받아 방에 입장하는 사용자
- 닉네임 입력
- 방장이 설정한 날짜 범위 안에서 본인의 가능한 날짜 입력
- 최종 후보 날짜와 다른 참여자들의 입력 현황 확인

## 3. 핵심 기능

### 3.1 첫 화면
- `초대코드 만들기` 버튼
- `초대코드 입력` 버튼

### 3.2 방 생성
- 사용자가 `초대코드 만들기` 버튼 클릭
- 시스템이 중복되지 않는 6자리 숫자 초대코드 생성
- 방장은 닉네임 입력
- 시작일과 종료일 입력으로 날짜 범위 설정
- 해당 범위 내에서 가능한 날짜 복수 선택
- 입력 완료 후 방 생성 완료

### 3.3 방 참여
- 사용자가 `초대코드 입력` 버튼 클릭
- 6자리 숫자 초대코드 입력
- 유효한 코드인지 검증
- 유효하면 닉네임 입력 화면으로 이동
- 방장이 설정한 날짜 범위 내에서 가능한 날짜 복수 선택
- 입력 완료 후 결과 화면으로 이동

### 3.4 결과 확인
- 전체 참여자 목록 확인
- 각 참여자의 가능한 날짜 확인
- 모두가 공통으로 가능한 날짜를 상단 후보로 노출
- 공통 날짜가 없으면 가장 많이 겹치는 날짜 순으로 추천 가능

## 4. 사용자 흐름

### 4.1 방장 흐름
1. 첫 화면 진입
2. `초대코드 만들기` 클릭
3. 시스템이 6자리 초대코드 생성
4. 닉네임 입력
5. 시작일/종료일 선택
6. 범위 내 가능한 날짜 선택
7. 방 생성 완료 화면 확인
8. 초대코드를 친구에게 공유
9. 참여자 입력이 완료되면 결과 확인

### 4.2 참여자 흐름
1. 첫 화면 진입
2. `초대코드 입력` 클릭
3. 6자리 초대코드 입력
4. 유효성 검증 통과
5. 닉네임 입력
6. 날짜 선택 화면 진입
7. 범위 내 가능한 날짜 선택
8. 제출 완료
9. 결과 화면에서 겹치는 날짜와 참여자별 가능 날짜 확인

## 5. 화면 기획

### 5.1 첫 화면
- 서비스 제목
- `초대코드 만들기` 버튼
- `초대코드 입력` 버튼

### 5.2 방 생성 화면
- 생성된 6자리 초대코드 표시
- 닉네임 입력 필드
- 시작일 선택
- 종료일 선택
- 날짜 캘린더 또는 날짜 리스트
- 가능한 날짜 복수 선택 UI
- 완료 버튼

### 5.3 초대코드 입력 화면
- 6자리 숫자 입력 필드
- 코드 확인 버튼
- 잘못된 코드일 경우 오류 메시지 표시

### 5.4 참여자 입력 화면
- 닉네임 입력 필드
- 방장이 설정한 날짜 범위 표시
- 가능한 날짜 복수 선택 UI
- 제출 버튼

### 5.5 결과 화면
- 공통 가능 날짜 후보 영역
- 참여자 수 표시
- 참여자별 가능한 날짜 목록
- 날짜별 가능 인원 수 표시

## 6. 데이터 구조 초안

### Room
- `id`: 내부 고유 ID
- `inviteCode`: 6자리 숫자 문자열
- `hostName`: 방장 닉네임
- `startDate`: 시작일
- `endDate`: 종료일
- `createdAt`: 생성일시
- `status`: 진행 상태

### Participant
- `id`: 내부 고유 ID
- `roomId`: 소속 방 ID
- `nickname`: 사용자 닉네임
- `isHost`: 방장 여부
- `availableDates`: 가능한 날짜 배열
- `joinedAt`: 참여 일시

## 7. 주요 정책

### 초대코드 정책
- 6자리 숫자만 허용
- 중복 코드 방지를 위한 서버 검증 필요
- 일정 시간이 지나면 만료시키는 정책 고려 가능

### 닉네임 정책
- 같은 방 안에서 중복 닉네임 허용 여부 결정 필요
- 초기 버전에서는 허용 가능하나 식별 편의상 비권장 안내 필요

### 날짜 선택 정책
- 시작일은 종료일보다 늦을 수 없음
- 사용자는 지정된 범위 밖 날짜를 선택할 수 없음
- 최소 1개 이상 날짜 선택 권장

## 8. 예외 상황

- 잘못된 초대코드 입력
- 이미 만료된 초대코드 입력
- 시작일/종료일 미입력
- 시작일이 종료일보다 뒤인 경우
- 가능한 날짜를 하나도 선택하지 않은 경우
- 같은 사용자가 새로고침 후 중복 제출하는 경우

## 9. MVP 범위

### 포함
- 방 생성
- 6자리 초대코드 발급
- 초대코드로 방 참여
- 닉네임 입력
- 날짜 범위 설정
- 범위 내 가능한 날짜 선택
- 공통 날짜 계산
- 참여자별 가능 날짜 조회

### 제외
- 로그인 기능
- 카카오톡/문자 공유 기능
- 실시간 알림
- 댓글/채팅 기능
- 시간대 단위 선택

## 10. 향후 확장 아이디어

- 날짜뿐 아니라 시간대까지 선택
- 가장 많이 겹치는 시간 자동 추천
- 방 링크 공유 기능
- 결과 이미지 저장 기능
- 마감 기능
- 방장만 결과 확정 가능하도록 권한 추가
- 실시간 반영 및 WebSocket 도입

## 11. 성공 기준

- 사용자가 1분 이내에 방을 만들 수 있어야 함
- 참여자는 초대코드 입력 후 30초 내 응답 가능해야 함
- 공통 날짜가 있으면 즉시 상단에 노출되어야 함
- 모바일에서 불편 없이 사용할 수 있어야 함
